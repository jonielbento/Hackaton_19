Arquitetura do Sistema de Agendamento Médico

Descrição da Arquitetura

O sistema de agendamento médico foi desenvolvido seguindo os princípios da arquitetura limpa (Clean Architecture) e padrão em camadas, proporcionando separação de responsabilidades e facilitando a manutenção e evolução do software.

Camadas da Aplicação

  • Camada de Apresentação (Hackaton.Web): Interface do usuário desenvolvida com Blazor WebAssembly, contendo páginas, componentes e serviços para comunicação com a API.
  • Camada de API (Hackaton.Api): Expõe endpoints RESTful para serem consumidos pela camada de apresentação, implementando controllers para cada entidade do sistema.
  • Camada de Aplicação (Hackaton.Application): Contém a lógica de negócio, serviços e DTOs (Data Transfer Objects) que fazem a ponte entre a API e o domínio.
  • Camada de Domínio (Hackaton.Domain): Núcleo da aplicação, contendo as entidades de negócio, interfaces e regras de domínio.
  • Camada de Infraestrutura (Hackaton.Infrastructure): Implementa o acesso a dados, repositórios e migrações do banco de dados.

Principais Entidades

  • Médico: Profissional de saúde com especialidade e valor de consulta.
  • Paciente: Usuário que agenda consultas médicas.
  • Consulta: Agendamento entre médico e paciente, com status e valor.
  • Agenda: Disponibilidade de horários do médico para consultas.

Fluxo de Dados

O fluxo de dados segue o princípio da dependência, onde as camadas externas dependem das camadas internas, mas nunca o contrário. A comunicação entre camadas é feita através de interfaces e DTOs, garantindo o baixo acoplamento.
